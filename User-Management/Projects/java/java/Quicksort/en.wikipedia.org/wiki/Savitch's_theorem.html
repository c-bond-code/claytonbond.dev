Savitch's theorem

From Wikipedia, the free encyclopedia
Jump to navigation Jump to search
Problems solvable nondeterministically in space S may be solved deterministically in space O(S²)

In computational complexity theory, Savitch's theorem, proved by Walter Savitch in 1970, gives a relationship between deterministic and non-deterministic space complexity. It states that for any function f ∈ Ω ( log ⁡ ( n ) ) {\displaystyle f\in \Omega (\log(n))} ,

N S P A C E ( f ( n ) ) ⊆ D S P A C E ( f ( n ) 2 ) . {\displaystyle {\mathsf {NSPACE}}\left(f\left(n\right)\right)\subseteq {\mathsf {DSPACE}}\left(f\left(n\right)^{2}\right).}

In other words, if a nondeterministic Turing machine can solve a problem using f ( n ) {\displaystyle f(n)} space, a deterministic Turing machine can solve the same problem in the square of that space bound.[1] Although it seems that nondeterminism may produce exponential gains in time, this theorem shows that it has a markedly more limited effect on space requirements.[2]

Proof[edit]

The proof relies on an algorithm for STCON, the problem of determining whether there is a path between two vertices in a directed graph, which runs in O ( ( log ⁡ n ) 2 ) {\displaystyle O\left((\log n)^{2}\right)} space for n {\displaystyle n} vertices. The basic idea of the algorithm is to solve recursively a somewhat more general problem, testing the existence of a path from a vertex s to another vertex t that uses at most k edges, where k is a parameter that is given as input to the recursive algorithm. STCON may be solved from this problem by setting k to n. To test for a k-edge path from s to t, one may test whether each vertex u may be the midpoint of the s-t path, by recursively searching for paths of half the length from s to u and u to t. Using pseudocode (in Python syntax) we can express this algorithm as follows: 

def k_edge_path(s, t, k) -> bool:
    """k initially equals n (which is the number of vertices)"""
    if k == 0:
        return s == t
    if k == 1:
        return (s, t) in edges
    for u in vertices:
        if k_edge_path(s, u, floor(k / 2)) and k_edge_path(u, t, ceil(k / 2)):
            return True
    return False

This search calls itself to a recursion depth of O ( log ⁡ n ) {\displaystyle O(\log n)} levels, each of which requires O ( log ⁡ n ) {\displaystyle O(\log n)} bits to store the function arguments and local variables at that level: k and all vertices (s, t, u) require O ( log ⁡ n ) {\displaystyle O(\log n)} bits each. The total auxiliary space complexity is thus O ( ( log ⁡ n ) 2 ) {\displaystyle O\left((\log n)^{2}\right)} .[Note 1] Although described above in the form of a program in a high-level language, the same algorithm may be implemented with the same asymptotic space bound on a Turing machine.

To see why this algorithm implies the theorem, consider the following. For any language L ∈ N S P A C E ( f ( n ) ) {\displaystyle L\in {\mathsf {NSPACE}}(f(n))} , there is a Turing machine M {\displaystyle M} which decides L {\displaystyle L} in space f ( n ) {\displaystyle f(n)} . Assume w.l.o.g. the alphabet is a binary alphabet { 0 , 1 } {\displaystyle \{0,1\}} (viz. L ⊆ { 0 , 1 } ∗ {\displaystyle L\subseteq \{0,1\}^{*}} ). For any input word x ∈ { 0 , 1 } ∗ {\displaystyle x\in \{0,1\}^{*}} , there is a directed graph G x M {\displaystyle G_{x}^{M}} whose vertices are the configurations of M {\displaystyle M} when running on the input x {\displaystyle x} . There may be infinitely many such configurations; e.g. when M {\displaystyle M} keeps writing a symbol on the tape and moving the head to the right in a loop, ad infinitum. The configurations then grow arbitrarily large. However, we know that at most f ( n ) {\displaystyle f\left(n\right)} space is needed to decide whether x ∈ L {\displaystyle x\in L} , so we only care about configurations of size at most f ( n ) {\displaystyle f\left(n\right)} ; call any such configuration admissible. There are finitely many admissible configurations; namely 2 O ( f ( n ) ) {\displaystyle 2^{O\left(f(n)\right)}} . Therefore, the induced subgraph [ G x M ] {\displaystyle [G_{x}^{M}]} of G x M {\displaystyle G_{x}^{M}} containing (exactly) the admissible configurations has 2 O ( f ( n ) ) {\displaystyle 2^{O\left(f(n)\right)}} vertices. For each input x ∈ { 0 , 1 } n {\displaystyle x\in \{0,1\}^{n}} , [ G x M ] {\displaystyle [G_{x}^{M}]} has a path from the starting configuration to an accepting configuration if and only if x ∈ L {\displaystyle x\in L} . Thus by deciding connectivity in [ G x M ] {\displaystyle [G_{x}^{M}]} , we can decide membership of x {\displaystyle x} in L {\displaystyle L} . By the above algorithm this can be done deterministically in space O ( ( log ⁡ 2 O ( f ( n ) ) ) 2 ) = O ( f ( n ) 2 ) {\displaystyle O\left(\left(\log 2^{O\left(f(n)\right)}\right)^{2}\right)=O\left(f\left(n\right)^{2}\right)} ; hence L {\displaystyle L} is in D S P A C E ( f ( n ) 2 ) {\displaystyle {\mathsf {DSPACE}}\left(f\left(n\right)^{2}\right)} .

Since this holds for all f ∈ Ω ( log ⁡ n ) {\displaystyle f\in \Omega (\log n)} and all L ∈ N S P A C E ( f ( n ) ) {\displaystyle L\in {\mathsf {NSPACE}}\left(f\left(n\right)\right)} , we get the statement of the theorem:

For all functions f ∈ Ω ( log ⁡ n ) {\displaystyle f\in \Omega (\log n)} , N S P A C E ( f ( n ) ) ⊆ D S P A C E ( f ( n ) 2 ) {\displaystyle {\mathsf {NSPACE}}\left(f\left(n\right)\right)\subseteq {\mathsf {DSPACE}}\left(f\left(n\right)^{2}\right)} .
  1. ^ Note there can be up to n 2 {\displaystyle n^{2}} edges in the input graph, each edge consuming 2 log ⁡ n {\displaystyle 2\log n} space, hence the total (i.e., not merely auxiliary) space complexity is O ( n 2 log ⁡ n ) {\displaystyle O(n^{2}\log n)} (and this bound is tight). However, edges may be represented implicitly, via the non-deterministic Turing machine under inspection.

Corollaries[edit]

Some important corollaries of the theorem include:

  • PSPACE = NPSPACE
    • This follows directly from the fact that the square of a polynomial function is still a polynomial function. It is believed that a similar relationship does not exist between the polynomial time complexity classes, P and NP, although this is still an open question.
  • NL ⊆ L2
    • STCON is NL-complete, and so all languages in NL are also in the complexity class L 2 = D S P A C E ( ( log ⁡ n ) 2 ) {\displaystyle {\mathsf {\color {Blue}L}}^{2}={\mathsf {DSPACE}}\left(\left(\log n\right)^{2}\right)} .
